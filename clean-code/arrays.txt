Первые три примера взял из реализации многоуровневого быстрого метода мультиполей 
для расчета задач электродинамики.

1. У каждой группы есть близкие группы, которые сейчас хранятся в обычном массиве.
Но они обрабатываются независимо, поэтому можно заменить массив на множество.

2. После чтения исходной сетки и построения дерева уровней у каждой группы имеется свой набор ребер,
который также можно хранить во множестве вместо обычного массива.

3. При реализации распараллеливания строится карта межпроцессорных обменов, сейчас реализованная также
при помощи массивов. В качестве альтернативы можно использовать ассоциативный массив.

4. В задаче реализации простой поисковой системы для пропуска стоп-слов используется массив
для их хранения. Предлагается использовать множество для хранения.

5. Нашел свою реализацию системы запуска задач для расчетов. Очередь сейчас реализована обычным массивом, 
из которого поочередно берутся и запускаются задачи. Более правильным будет использование структуры очереди.
