При работе с моделированием физических процессов необходимо
учитывать множество параметров и констант, поэтому примеры взял из рабочего проекта.

1. Связывание во время компиляции программы.
Пример - число точек квадратуры для интегрирования NUMBER_QUADRATURE_POINTS
задается как константа в самом начале программы. Выбор в пользу связывания
на этапе компиляции был сделан по двум причинам:
    1. Этот параметр относится только к внутренним алгоритмам, и нет смысла его записывать в файл конфигурации.
    2. Частое обращение внутри счетных алгоритмов.

2. Связываение на этапе выполнения.
Пример - имя файла с расчетной сеткой surfaceMeshFile задается следующим образом:

surfaceMeshName = GetSurfaceMeshName().
Выбор был сделан по следующим причинам:
    1. Этот метод вызывается всего один раз в начале программы.
    2. Связываение на этапе выполнения позволяет запускать множество расчетов, не делая пересборку
    проекта, достаточно просто менять файлы конфигурации.

3. Связывание на этапе выполнения.
Пример - задание директории для хранения медиафайлов.

BASE_DIR = ... // задает корневую директорию проекта
MEDIA_ROOT = os.path.join(BASE_DIR, 'media')

Был выбран вариант связывания на этапе выполнения по причине:
    1. Переменная MEDIA_ROOT будет содержать в себе путь до папки media, 
    в зависимости от того, где будет размещен проект, чего не будет, если
    записать в неё абсолятный путь.
